### `01` → **DocSearch** — Hybrid Retrieval Engine

```yaml
Type: Information Retrieval System
Status: Production-Ready
Architecture: TF-IDF + FAISS HNSW + Cross-Encoder Reranking
```

**Engineering Depth:**
- Offline multilingual document search (English, Hindi, Telugu)
- Hybrid fusion via Reciprocal Rank Fusion — balances keyword precision with semantic recall
- HNSW approximate nearest neighbor optimized for CPU inference
- OCR pipeline (PDFium + Tesseract) for scanned document handling
- Semantic chunking to reduce false positives from document noise

**Why It Matters:**  
Local document search remains unsolved for real-world collections with incomplete metadata, mixed languages, and noisy OCR. This brings production IR techniques to offline workflows — privacy-first, CPU-friendly, and engineered for messy data.

**Metrics:** `MRR: 0.7556` `MAP: 0.7052` `Precision@5: 0.24`

### `02` → **Speaker Intelligence System** — Multilingual Audio Pipeline

```yaml
Type: NLP + Speech Processing System
Status: Production-Grade
Pipeline: Diarization → Identification → Transcription → Translation
```

**Engineering Depth:**
- Multi-stage audio intelligence across Hindi, English, Punjabi
- Speaker diarization + voice identification with SpeechBrain
- ASR transcription via Whisper with noise robustness
- Multilingual translation pipeline using IndicTrans2
- Modular architecture for real-world audio sources

**Why It Matters:**  
Multilingual audio processing for Indian languages is complex — accent variation, code-switching, and domain-specific vocabulary break most off-the-shelf systems. This pipeline handles real-world audio at scale.

`[PyTorch • SpeechBrain • Whisper • IndicTrans2 • NLP]`
